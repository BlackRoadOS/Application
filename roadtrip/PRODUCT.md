## Memory Engine

### Tiered Architecture
- **Hot Layer**: recent conversations, active projects, core preferences. Sub-second retrieval.
- **Warm Layer**: consolidated insights, relationship graphs, learned patterns. Built during consolidation.
- **Cold Layer**: full raw history, audit trails. Searchable, exportable via OneWay.

### Dynamic Movement
Frequently referenced → Hot. Valuable but rare → Warm. Low-signal → Cold. Every movement on RoadChain.

### Road Log Interface
Hot = bright road lines. Warm = steady highway. Cold = archived side roads. One-click promote/demote/protect/delete.

---

## Memory Consolidation

### Triggers
Automatic (milestones, scheduled, size thresholds) or manual from Training Log.

### Pipeline
**A. Summarization** — Three depths (micro, mid-level, context-rich). Agent collaboration: Cecilia (structure), Calliope (language), Alexandria (facts), Sophia (wisdom framing). Consensus before finalizing.

**B. Merging** — Similar conversations merged into stronger fact nodes.

**C. Relationship Graphs** — Links across products ("pricing discussion → RoadWork invoice → RoadView feedback").

**D. Preference Extraction** — Behavioral patterns identified and stored.

**E. Selective Decay** — Low-relevance gracefully to Cold. Never deleted unless chosen.

**F. Reflective Integration** — Higher-level insights generated by Lucidia and Sophia.

### User Control
Consolidation Preview in Training Log. Approve, edit, reject. Full history on RoadChain.
